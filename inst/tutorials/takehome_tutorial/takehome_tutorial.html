Creating Functions

In the code block below, create a function titled ‘count_unique’ that will count the number of unique values for a specified column.

You should use the following steps:

  1. Assign a variable that contains all unique values for that column with the unique() function.
  2. Use the length() function to count the number of values in the variable you created in step 1.
count_unique <- function(col_name) {
  unique_values <- unique(col_name)
  
  length(unique_values)
}

Now create a second function titled ‘count_na’ that calculates the number of NA’s in each column.

First, use the is.na() function around a specific column. Second, use the sum() function to count the number of values that return TRUE. (If the logical value returned is TRUE, that means there is an NA and therefore we should count it.)

count_na <- function(col_name) {
  sum(is.na(col_name))
}

Multiple tests

If you give filter() more than one logical test, filter() will combine the tests with an implied “and.”In other words, filter() will return only the rows that return TRUE for every test. You can combine tests in other ways with Boolean operators…

Boolean operators

&, |, and !

R uses boolean operators to combine multiple logical comparisons into a single logical test. These include & (and), | (or), ! (not or negation), and xor() (exactly or).

Both | and xor() will return TRUE is one or the other logical comparison returns TRUE. xor() differs from | in that it will return FALSE if both logical comparisons return TRUE. The name xor stands for exactly or.
